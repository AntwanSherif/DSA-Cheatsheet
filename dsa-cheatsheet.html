Two Pointers
Eliminate O(n²) nested loops by moving two indices through an array simultaneously.

When to use

  • Pair summing to a target (sorted)
    Given a sorted array, find two elements that add up to a target value.
    Signal: a "sorted array" together with "find two numbers / a pair that sum to / add up to"
    e.g. Two Sum II — Input Array Is Sorted
  • Palindrome check
    Verify a string or array reads the same forward and backward.
    Signal: "is a palindrome", "reads the same backward", "ignoring non-alphanumeric"
    e.g. Valid Palindrome
  • Merge / compare two sorted sequences in place
    Advance one pointer per sorted array; when merging in place (e.g. LC 88), fill from the back of the destination so unread values are never overwritten.
    Signal: "two sorted arrays", "merge in-place", "without using extra space"
    e.g. Merge Sorted Array
  • In-place array rewrite (fast / slow pointer)
    Overwrite an array in one pass so kept elements move to the front; a slow pointer marks the write boundary while a fast pointer scans.
    Signal: "in-place", "remove duplicates", "move zeroes", "return the new length"
    e.g. Remove Duplicates from Sorted Array
  • Maximize over a shrinking range
    Pointers start at both ends; move the limiting one inward to optimize an area or quantity.
    Signal: "maximize an area / quantity bounded by two ends", opposite-end pointers where you move the limiting side inward
    e.g. Container With Most Water

How it works — Two Sum II

arr = [ 1, 3, 5, 7, 11 ] target = 10 indices = 0 1 2 3 4 Rule: sum < target → L++ (left value too small — move L right) sum > target → R-- (right value too big — move R left) sum == target → return [L+1, R+1]
① ──────────────────────────────────── [L=0] [R=4] sum = 1 + 11 = 12 > 10 → R--
① Compute sum = arr[L] + arr[R] = 1 + 11 = 12. 12 is greater than the target, so the right value is too big — move R left by one. (The loop runs while L < R.)
② ──────────────────────────────────── [L=0] [R=3] sum = 1 + 7 = 8 < 10 → L++
② The new R gives sum = 1 + 7 = 8, now below the target, so the left value is too small — move L right by one.
③ ──────────────────────────────────── [L=1] [R=3] sum = 3 + 7 = 10 == 10 ✓
③ sum = 3 + 7 = 10 equals the target. The pointers are at 0-indexed [1, 3]; this problem returns 1-indexed positions, so return [2, 4]. If no pair ever matches, the loop ends once L ≥ R and returns [].

Code Template (TypeScript)

Two Sum II – Input Array Is Sorted Easy
Given a 1-indexed sorted array, return the 1-indexed positions of two numbers that add up to target. Exactly one solution exists.
function twoSumSorted(nums: number[], target: number): number[] {
  let left = 0, right = nums.length - 1;

  while (left < right) {
    const sum = nums[left] + nums[right];
    if (sum === target) return [left + 1, right + 1]; // 1-indexed
    else if (sum < target) left++;   // need larger sum → move left pointer right
    else right--;                     // need smaller sum → move right pointer left (sorted array guarantees this works)
  }

  return [];
}
Remove Duplicates from Sorted Array Easy
Modify the array in-place so each unique value appears once. Return the number of unique elements. Relative order must be preserved.
function removeDuplicates(nums: number[]): number {
  let slow = 0; // slow: boundary of the unique-values region
  for (let fast = 1; fast < nums.length; fast++) {
    if (nums[fast] !== nums[slow]) {
      slow++;             // expand unique region
      nums[slow] = nums[fast]; // write next unique value
    }
  }
  return slow + 1; // count of unique elements
}

Sliding Window
Maintain a contiguous window; expand right, shrink left when a constraint is violated.

When to use

  • Longest/shortest subarray or substring satisfying a condition
    Given an array or string, find the longest (or shortest) contiguous subarray/substring that obeys a constraint such as "at most k distinct characters" or "sum at most S".
    Signal: "longest subarray", "shortest subarray", "contiguous", "satisfying a condition", "at most k"
    e.g. Minimum Size Subarray Sum
  • Maximum sum of a fixed-size window
    Slide a window of exactly k elements across the array; add the entering element and subtract the leaving element each step instead of recomputing the whole sum.
    Signal: "subarray of length k", "fixed window", "maximum average subarray"
    e.g. Maximum Average Subarray I
  • Minimum window containing all required characters
    Expand the right pointer until the window contains every required element, then shrink the left pointer to minimise the window size while maintaining coverage.
    Signal: "minimum window", "contains all characters", "smallest substring that includes"
    e.g. Minimum Window Substring
  • Substring without repeating characters
    Maintain a window where no character appears twice; when a duplicate is encountered, advance the left pointer past the previous occurrence.
    Signal: "no repeating characters", "all unique characters", "longest substring without"
    e.g. Longest Substring Without Repeating Characters

How it works — Max sum window (k=3) & variable shrink

FIXED WINDOW — arr = [ 2, 1, 5, 1, 3, 2 ] k = 3 indices = 0 1 2 3 4 5 Rule: slide right one step each time — add arr[R], subtract arr[R-k], update max. VARIABLE WINDOW — s = "abcba" (longest substring, no repeating chars) Rule: expand R each step; when duplicate detected → advance L past last occurrence.
① ──────────────────────────────────── FIXED: [L=0 → R=2] window=[2,1,5] sum=8 max=8
① Seed the first window by summing arr[0..k-1] = 2+1+5 = 8. This is the starting max.
② ──────────────────────────────────── FIXED: [L=1 → R=3] window=[1,5,1] sum=7 max=8 FIXED: [L=2 → R=4] window=[5,1,3] sum=9 max=9 ← updated FIXED: [L=3 → R=5] window=[1,3,2] sum=6 max=9 ← final
② Each slide: windowSum += arr[R] - arr[R-k]. At R=3 the sum is 8 + arr[3] − arr[0] = 8 + 1 − 2 = 7, max stays 8. At R=4 the sum rises to 9 — new max. At R=5 sum drops to 6, max stays 9. Return 9.
③ ──────────────────────────────────── VARIABLE: [L=0, R=0] window="a" seen={a:0} len=1 [L=0, R=1] window="ab" seen={a:0,b:1} len=2 [L=0, R=2] window="abc" seen={a:0,b:1,c:2} len=3 ← max
③ Variable window: expand R one character at a time. While no duplicate exists in the window, update maxLen = R - L + 1. After "abc", maxLen = 3.
④ ──────────────────────────────────── VARIABLE: R=3, ch='b' — duplicate! lastSeen[b]=1 ≥ L=0 → L = 1+1 = 2 [L=2, R=3] window="cb" seen={...,b:3} len=2
④ At R=3 the character 'b' was already seen at index 1, which is inside the current window (index 1 ≥ L=0). Shrink: jump L to lastSeen['b']+1 = 2. Window is now "cb"; maxLen stays 3.
⑤ ──────────────────────────────────── VARIABLE: R=4, ch='a' — lastSeen[a]=0 < L=2 → no shrink (outside window) [L=2, R=4] window="cba" len=3 maxLen=3 (unchanged)
⑤ At R=4 'a' is in the map but its last index (0) is to the left of L (2) — it is not inside the current window — so no shrink. Window "cba" has length 3. Loop ends; return maxLen = 3. If the string were empty, the loop never runs and 0 is returned.

Code Template (TypeScript)

Longest Substring Without Repeating Characters Medium
Given a string, find the length of the longest substring that contains no repeating characters.
function lengthOfLongestSubstring(s: string): number {
  const lastSeen = new Map<string, number>(); // char → last index seen
  let left = 0, maxLen = 0;

  for (let right = 0; right < s.length; right++) {
    const ch = s[right];
    // `lastSeen.get(ch)! >= left` guards against chars seen before the current window:
    // the char may be in the map but to the left of `left`, so it's not a duplicate in the window.
    if (lastSeen.has(ch) && lastSeen.get(ch)! >= left) {
      left = lastSeen.get(ch)! + 1; // shrink window: jump left past the duplicate
    }
    lastSeen.set(ch, right);
    maxLen = Math.max(maxLen, right - left + 1); // window invariant: no duplicates in [left, right]
  }

  return maxLen;
}
Maximum Average Subarray I Easy
Given an integer array and integer k, find the contiguous subarray of length k with the maximum average value.
function maxSumFixed(nums: number[], k: number): number {
  let windowSum = nums.slice(0, k).reduce((a, b) => a + b, 0);
  let max = windowSum;

  for (let i = k; i < nums.length; i++) {
    windowSum += nums[i] - nums[i - k]; // slide: add element entering window, subtract element leaving
    max = Math.max(max, windowSum);
  }

  return max; // divide by k for average
}

Prefix Sums
Precompute cumulative sums so any range sum query is answered in O(1).

When to use

  • Range sum queries on a static array
    Precompute a prefix array once so that any sum over a contiguous subarray can be answered in O(1) by subtracting two prefix values.
    Signal: "sum of elements from i to j", "range sum query", "multiple queries on static array"
    e.g. Range Sum Query – Immutable
  • Count subarrays whose sum equals k
    Track a running prefix sum in a hashmap; if (currentSum − k) has been seen before, those past prefixes form subarrays ending here that sum exactly to k.
    Signal: "number of subarrays", "sum equals k", "count continuous subarrays"
    e.g. Subarray Sum Equals K
  • Find if a subarray with a given property exists
    Store prefix sums (or parity/mod values) in a set; a matching value found later confirms the desired subarray exists between the two positions.
    Signal: "subarray sum equals k", "count subarrays divisible by k", a cumulative-sum remainder seen before
    e.g. Continuous Subarray Sum
  • 2D matrix region sum queries
    Extend the prefix idea to two dimensions: prefix2D[r][c] stores the sum of the rectangle from (0,0) to (r-1,c-1), enabling any rectangular region sum in O(1).
    Signal: "sum of rectangle", "region sum", "matrix subregion", "2D range query"
    e.g. Range Sum Query 2D – Immutable

How it works — Subarray Sum / range query

arr = [ 1, 2, 3, 4, 5 ] index = 0 1 2 3 4 Build prefix (size n+1): prefix[0] = 0, prefix[i+1] = prefix[i] + arr[i] prefix = [ 0, 1, 3, 6, 10, 15 ] index = 0 1 2 3 4 5 Range query formula: rangeSum(i, j) = prefix[j+1] - prefix[i] Subarray-sum-k rule: for each index, if (currentSum - k) is in the map → add its count.
① ──────────────────────────────────── Build prefix: prefix[0]=0 prefix[1]=0+1=1 prefix[2]=1+2=3 prefix[3]=3+3=6 prefix[4]=6+4=10 prefix[5]=10+5=15
① Initialise prefix[0] = 0 (empty prefix). Then iterate: prefix[i+1] = prefix[i] + arr[i]. The full prefix array is [0, 1, 3, 6, 10, 15]. This O(n) build is done once.
② ──────────────────────────────────── Range query: rangeSum(i=1, j=3) = prefix[j+1] - prefix[i] = prefix[4] - prefix[1] = 10 - 1 = 9 ✓ (arr[1]+arr[2]+arr[3] = 2+3+4 = 9)
② To sum elements from index 1 to 3 inclusive, subtract the prefix up to i from the prefix up to j+1. The operation is O(1) regardless of how many queries follow.
③ ──────────────────────────────────── Subarray sum = k (k=5), arr=[1,2,3,4,5] Seed map = { 0:1 } currentSum=0 i=0: currentSum=1 need=1-5=-4 not in map → map={0:1, 1:1} i=1: currentSum=3 need=3-5=-2 not in map → map={0:1,1:1,3:1} i=2: currentSum=6 need=6-5=1 found map[1]=1 → result += 1 map={0:1,1:1,3:1,6:1} i=3: currentSum=10 need=10-5=5 not in map → map={...,10:1} i=4: currentSum=15 need=15-5=10 found map[10]=1 → result += 1
③ Seed the hashmap with {0:1} so subarrays starting at index 0 are handled. At each element add it to currentSum, then check whether (currentSum - k) was a previous prefix sum. At i=2 (currentSum=6), need = 6-5 = 1 which exists in the map — the subarray arr[1..2] = [2, 3] sums to 5. At i=4 (currentSum=15), need = 15-5 = 10 which exists — the subarray arr[4] = [5] sums to 5. Total count = 2. If the array is empty no iterations run and 0 is returned. The {0:1} seed is critical: it lets a prefix sum that itself equals k match immediately (currentSum - k = 0 is found on the very first hit), covering subarrays that start at index 0.

Code Template (TypeScript)

Range Sum Query – Immutable Easy
Given an integer array, handle multiple queries sumRange(i, j) — return the sum of elements between indices i and j inclusive. Preprocess once for O(1) queries.
// Build once: prefix[i] = sum of all elements before index i
function buildPrefix(nums: number[]): number[] {
  const prefix = [0]; // prefix[0] = 0 (empty prefix)
  for (const n of nums) {
    prefix.push(prefix[prefix.length - 1] + n);
  }
  return prefix;
}

// Query in O(1): sum(i..j) = prefix[j+1] - prefix[i]
function rangeSum(prefix: number[], i: number, j: number): number {
  return prefix[j + 1] - prefix[i];
}
Subarray Sum Equals K Medium
Given an array of integers (can be negative) and an integer k, return the total number of subarrays whose elements sum to k.
function subarraySum(nums: number[], k: number): number {
  // Seed with {0: 1}: the empty prefix has sum 0, and it "exists" once.
  // This handles subarrays starting from index 0.
  const prefixCounts = new Map([[0, 1]]); // prefixSum → frequency
  let currentSum = 0, result = 0;

  for (const n of nums) {
    currentSum += n;
    // If (currentSum - k) was a previous prefix sum, those prefixes
    // form subarrays ending here that sum exactly to k.
    result += prefixCounts.get(currentSum - k) ?? 0;
    prefixCounts.set(currentSum, (prefixCounts.get(currentSum) ?? 0) + 1);
  }

  return result;
}

Hash Maps & Sets
O(1) average insert/lookup. The single most-used tool to eliminate nested loops.

When to use

  • Count element frequencies
    Iterate once, incrementing each element's count in a map; then query the map for majority elements, anagram comparisons, or character distributions.
    Signal: "frequency", "count occurrences", "most common", "anagram", "character count"
    e.g. Valid Anagram
  • "Have I seen this before?" — existence check in O(1)
    Add elements to a Set as you go; a hit on has() means the element appeared earlier, turning an O(n²) scan into a single pass.
    Signal: "contains duplicate", "already visited", "seen before", "detect duplicate"
    e.g. Contains Duplicate
  • Find complement: target − current (Two Sum)
    Store each value with its index as you iterate; before inserting, look up whether the complement (target − current) already exists in the map.
    Signal: "two numbers that add up to", "find indices such that", "complement", "pair summing to target"
    e.g. Two Sum
  • Group elements by a computed key (anagrams)
    Derive a canonical key for each element (e.g. sort its characters) and accumulate into a map of key → list; elements that share a key are in the same group.
    Signal: "group anagrams", "group by", "cluster", "canonical form", "sort characters"
    e.g. Group Anagrams
  • Deduplicate or track unique elements
    Load all elements into a Set; membership checks then run in O(1), enabling algorithms like finding the start of a consecutive sequence without repeated inner loops.
    Signal: "unique", "deduplicate", "no repeats", "longest consecutive sequence"
    e.g. Longest Consecutive Sequence

How it works — Three core hash-map patterns

Three independent patterns; each is a self-contained trace. PATTERN 1 — Frequency count: s = "aabbc" PATTERN 2 — Complement lookup (Two Sum): nums=[2,7,11,15] target=9 PATTERN 3 — Grouping by key (Group Anagrams): words=["eat","tea","tan","ate","nat","bat"]
① ──────────────────────────────────── PATTERN 1 — Frequency count ch='a': freq={a:1} ch='a': freq={a:2} ch='b': freq={a:2,b:1} ch='b': freq={a:2,b:2} ch='c': freq={a:2,b:2,c:1}
① Scan the string once. For each character, increment its count in the map (default 0 if unseen: freq.get(ch) ?? 0). After the loop the map holds every character's frequency. Use this to check anagrams (compare two frequency maps), find the majority element, or compare character distributions — all in O(n) time and O(k) space where k is the alphabet size.
② ──────────────────────────────────── PATTERN 2 — Complement lookup seen = {} i=0: nums[0]=2 need=9-2=7 not in seen → seen={2:0} i=1: nums[1]=7 need=9-7=2 found seen[2]=0 → return [0,1] ✓
② Check the complement BEFORE inserting the current value. If we inserted first we could accidentally match an element with itself. At i=1, need = 9-7 = 2 was stored at index 0, so return [0, 1]. If no pair exists the loop finishes without a match. This pattern generalises to any "find two things that satisfy a relationship" problem.
③ ──────────────────────────────────── PATTERN 3 — Grouping by key "eat" → sort → "aet" : groups={"aet":["eat"]} "tea" → sort → "aet" : groups={"aet":["eat","tea"]} "tan" → sort → "ant" : groups={"aet":[...],"ant":["tan"]} "ate" → sort → "aet" : groups={"aet":["eat","tea","ate"],...} "nat" → sort → "ant" : groups={...,"ant":["tan","nat"]} "bat" → sort → "abt" : groups={...,"abt":["bat"]} result → [["eat","tea","ate"],["tan","nat"],["bat"]]
③ Derive a canonical key that is identical for all equivalent elements — here, sorting a word's characters. Use the key to look up (or create) the group list in the map, then append the original word. Any element-to-key mapping works: character-frequency tuples, mod values, sorted digits, etc. Return all map values at the end. If the input array is empty the map stays empty and an empty list is returned.

Two Sum Easy
Given an array of integers and a target, return the indices of the two numbers that add up to the target. Each input has exactly one solution.
function twoSum(nums: number[], target: number): number[] {
  const seen = new Map<number, number>(); // value → index
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    // Check BEFORE inserting: if we inserted first, we'd risk matching the element with itself.
    if (seen.has(complement)) return [seen.get(complement)!, i];
    seen.set(nums[i], i);
  }
  return [];
}
Group Anagrams Medium
Given an array of strings, group the anagrams together and return the groups in any order.
function groupAnagrams(strs: string[]): string[][] {
  const groups = new Map<string, string[]>();
  for (const word of strs) {
    // Sorting the characters gives a canonical key shared by all anagrams.
    const key = [...word].sort().join('');
    if (!groups.has(key)) groups.set(key, []);
    groups.get(key)!.push(word);
  }
  return [...groups.values()];
}

// Bonus — Set dedup / existence check:
const seen = new Set<number>();
for (const n of nums) {
  if (seen.has(n)) return true; // duplicate found in O(1)
  seen.add(n);
}

Fast & Slow Pointers
Two pointers at different speeds: slow moves 1 step, fast moves 2. They meet inside a cycle.

When to use

  • Cycle detection in a linked list
    Determine whether a linked list contains a loop — a node whose next pointer points back to an earlier node rather than null.
    Signal: "linked list", "cycle", "loop", "detect if a list is circular"
    e.g. Linked List Cycle
  • Find the middle of a linked list
    Locate the midpoint in one pass without knowing the length — useful as a sub-step when reversing a half or merging.
    Signal: "middle node", "split the list in half", "find the median node"
    e.g. Middle of the Linked List
  • Find the start of a cycle
    After detecting a cycle, locate the exact node where it begins using a second phase of Floyd's algorithm.
    Signal: "return the node where the cycle begins", "find cycle entrance", "detect and locate cycle"
    e.g. Linked List Cycle II
  • Abstract cycle detection (Happy Number)
    Any sequence defined by a "next" function that may loop can be treated as an implicit linked list — apply fast/slow to detect if the sequence cycles.
    Signal: "repeatedly apply a transformation", "will it ever reach X", "does it loop forever"
    e.g. Happy Number

How it works — Cycle detection (Floyd's)

List A (no cycle): 1→2→3→4→5→null List B (cycle): 1→2→3→4→5→6→3 (node 6 points back to node 3) slow moves 1 step per iteration; fast moves 2 steps. Rule: if fast or fast.next reaches null → no cycle → return false. if slow === fast → cycle detected → return true.
① ──────────────────────────────────── NO-CYCLE BRANCH (List A: 1→2→3→4→5→null) start: slow=1, fast=1 step 1: slow=2, fast=3 step 2: slow=3, fast=5 step 3 check: fast.next is null → while-condition fails → exit (slow stays put, no advance)
① When there is no cycle fast eventually reaches the end of the list. After step 2 fast is at node 5; fast.next is null, so the while-condition fast !== null && fast.next !== null fails and the loop exits. We return false — no meeting point was ever found.
② ──────────────────────────────────── CYCLE BRANCH (List B: 1→2→3→4→5→6→3) start: slow=1, fast=1 step 1: slow=2, fast=3 step 2: slow=3, fast=5 step 3: slow=4, fast=3 (fast wrapped: 5→6→3) step 4: slow=5, fast=5 ← slow === fast → cycle!
② Inside a cycle fast "laps" slow: fast moves two nodes per step while slow moves one, so within the cycle, fast closes the gap on slow by one node per iteration (fast moves 2, slow moves 1), so they are guaranteed to meet. They must eventually land on the same node. Here they meet at node 5 after step 4. The moment slow === fast we return true. The meeting point is somewhere inside the cycle — not necessarily the cycle entrance.
③ ──────────────────────────────────── FIND-MIDDLE BONUS (List A: 1→2→3→4→5) start: slow=1, fast=1 step 1: slow=2, fast=3 step 2: slow=3, fast=5 ← fast.next is null → stop slow=3 is the middle node ✓
③ A second application: to find the middle in one pass, stop when fast.next or fast.next.next is null. Because fast travels at 2× speed, when fast reaches the last node slow is exactly at the midpoint. For an even-length list (e.g. 1→2→3→4) this condition fires when fast=3, leaving slow at node 2 — the second of the two middle nodes, which is what LC 876 requires.

Code Template (TypeScript)

Linked List Cycle Easy
Given the head of a linked list, determine if the list contains a cycle.
class ListNode {
  val: number;
  next: ListNode | null = null;
  constructor(val: number) { this.val = val; }
}

// Floyd's cycle detection: fast moves 2× speed.
// In a cycle, fast "laps" slow — they must meet inside the cycle.
// No cycle → fast reaches null first.
function hasCycle(head: ListNode | null): boolean {
  let slow = head, fast = head;
  while (fast !== null && fast.next !== null) {
    slow = slow!.next;
    fast = fast.next.next;
    if (slow === fast) return true;
  }
  return false;
}
Middle of the Linked List Easy
Given the head of a singly linked list, return the middle node. If two middle nodes exist, return the second one.
function middleNode(head: ListNode): ListNode {
  let slow: ListNode = head, fast: ListNode = head;
  // Terminate when fast.next or fast.next.next is null.
  // For even-length lists this leaves slow at the SECOND middle (as required).
  while (fast.next !== null && fast.next.next !== null) {
    slow = slow.next!;
    fast = fast.next.next;
  }
  return slow;
}
Happy Number Easy
Starting from n, repeatedly replace the number with the sum of squares of its digits. Return true if the process eventually reaches 1; false if it loops forever.
// Unhappy numbers always enter a cycle — this is a cycle-detection problem.
// Apply Floyd's algorithm on the abstract "next" function (digit-square-sum).
function isHappy(n: number): boolean {
  const digitSquareSum = (x: number) =>
    String(x).split('').reduce((s, d) => s + Number(d) ** 2, 0);
  let slow = n, fast = n;
  do {
    slow = digitSquareSum(slow);                         // 1 step
    fast = digitSquareSum(digitSquareSum(fast));          // 2 steps
  } while (slow !== fast);
  return slow === 1; // met at 1 → happy; met elsewhere → cycle → not happy
}

Linked List Reversal
Reverse a linked list by re-pointing next pointers one node at a time with three variables: prev, curr, next.

When to use

  • Reverse an entire linked list
    Re-point every node's next pointer so the list reads in the opposite direction, returning the former tail as the new head.
    Signal: "reverse a linked list", "return the new head", "in-place reversal"
    e.g. Reverse Linked List
  • Reverse a sublist (positions m to n)
    Walk to the sublist boundary, reverse only that portion, then reconnect the modified section to the untouched parts.
    Signal: "reverse between positions", "reverse from left to right", "reverse a portion of the list"
    e.g. Reverse Linked List II
  • Check if a linked list is a palindrome
    Find the middle with fast/slow pointers, reverse the second half in-place, then compare it to the first half node by node.
    Signal: "palindrome linked list", "is the list the same forwards and backwards"
    e.g. Palindrome Linked List
  • Reorder list (interleave halves)
    Split the list at the middle, reverse the second half, then merge the two halves by alternating nodes from each.
    Signal: "reorder to L0→Ln→L1→Ln-1…", "interleave first and last nodes"
    e.g. Reorder List

How it works — Reverse a linked list

Input list: 1→2→3→null (head = node 1) Three variables track state each iteration: prev — the already-reversed portion's new head (starts null) curr — the node being processed this step next — saved copy of curr.next before we overwrite it Loop invariant: at start of each iteration, nodes before curr are fully reversed.
① ──────────────────────────────────── Before: prev=null curr=1 (1→2→3→null) next = curr.next → next=2 curr.next = prev → 1→null prev = curr → prev=1 curr = next → curr=2 After: null←1 | curr=2→3→null
① Save curr.next before overwriting it (otherwise we lose the rest of the list). Flip curr.next to point at prev — node 1 now points to null, reversing its link. Advance prev to the current node (prev = 1) and curr to the saved next (curr = 2). The left portion null←1 is fully reversed; the right portion 2→3→null is untouched.
② ──────────────────────────────────── Before: prev=1 curr=2 (2→3→null) next = curr.next → next=3 curr.next = prev → 2→1→null prev = curr → prev=2 curr = next → curr=3 After: null←1←2 | curr=3→null
② The same four operations apply: save next, flip the pointer, advance prev, advance curr. Node 2 now points back to node 1, extending the reversed portion. The reversed chain so far is null←1←2; one node (3) remains to be processed.
③ ──────────────────────────────────── Before: prev=2 curr=3 (3→null) next = curr.next → next=null curr.next = prev → 3→2→1→null prev = curr → prev=3 curr = next → curr=null ← loop ends Final: null←1←2←3 new head = prev = 3
③ Node 3's original next was already null, so saving it is harmless. After flipping, 3→2→1→null completes the reversal. curr becomes null, so the while-condition fails and the loop exits. prev points to node 3, which is the new head — return prev. If the input list is empty (head = null), curr starts as null, the loop never runs, and prev = null is returned correctly.

Reverse Linked List Easy
Given the head of a singly linked list, reverse the list in-place and return the new head.
function reverseList(head: ListNode | null): ListNode | null {
  let prev: ListNode | null = null;
  let curr = head;

  while (curr !== null) {
    const next = curr.next; // 1. save next (we're about to overwrite curr.next)
    curr.next = prev;       // 2. reverse the pointer
    prev = curr;            // 3. advance prev to current node
    curr = next;            // 4. advance curr to saved next
  }

  return prev; // prev is now the new head (last node of original list)
}
Reverse Linked List II Medium
Given the head of a singly linked list and two integers left and right, reverse the nodes from position left to right (1-indexed) and return the modified list.
function reverseBetween(head: ListNode | null, left: number, right: number): ListNode | null {
  // Dummy node lets us handle the edge case where left=1 without special-casing the head.
  const dummy = new ListNode(0);
  dummy.next = head;
  let pre: ListNode = dummy;

  for (let i = 0; i < left - 1; i++) pre = pre.next!; // walk to node before sublist

  let curr = pre.next!;
  // In-place insertion: each iteration pulls curr.next to the front of the sublist.
  for (let i = 0; i < right - left; i++) {
    const next = curr.next!;
    curr.next = next.next;  // unlink `next` from its position
    next.next = pre.next;   // insert `next` at front of sublist
    pre.next = next;
  }

  return dummy.next;
}

How it works — Tree DFS — pre / in / post order

Tree used in all three blocks: 4 ← root / \ 2 6 / \ / \ 1 3 5 7 Rule for every traversal: if node is null, return immediately (base case). The only difference is WHEN the current node is "visited" (its value recorded) relative to the left and right recursive calls.
① ──────────────────────────────────── PREORDER (Root → Left → Right) visit(4) → recurse left visit(2) → recurse left visit(1) → left=null, right=null → return recurse right visit(3) → left=null, right=null → return recurse right visit(6) → recurse left visit(5) → left=null, right=null → return recurse right visit(7) → left=null, right=null → return Result: 4, 2, 1, 3, 6, 5, 7
① In preorder the current node is recorded BEFORE recursing into either child. The recursive rule is: visit node → recurse left → recurse right. This visits the root first, then explores the entire left subtree, then the entire right subtree. Preorder output (4,2,1,3,6,5,7) reconstructs the tree unambiguously — useful for serialization and copying.
② ──────────────────────────────────── INORDER (Left → Root → Right) recurse left of 4 recurse left of 2 recurse left of 1 → null → return visit(1) recurse right of 1 → null → return visit(2) recurse right of 2 visit(3) visit(4) recurse right of 4 recurse left of 6 recurse left of 5 → null → return visit(5) recurse right of 5 → null → return visit(6) recurse right of 6 recurse left of 7 → null → return visit(7) recurse right of 7 → null → return Result: 1, 2, 3, 4, 5, 6, 7
② In inorder the current node is recorded BETWEEN the two recursive calls: recurse left → visit node → recurse right. For any valid BST this ordering always produces the values in ascending sorted order (1,2,3,4,5,6,7 here). Use inorder to find the kth smallest element, validate a BST, or extract a sorted sequence without sorting.
③ ──────────────────────────────────── POSTORDER (Left → Right → Root) recurse left of 4 recurse left of 2 visit(1) recurse right of 2 visit(3) visit(2) recurse right of 4 recurse left of 6 visit(5) recurse right of 6 visit(7) visit(6) visit(4) Result: 1, 3, 2, 5, 7, 6, 4
③ In postorder the current node is recorded AFTER both children have been fully processed: recurse left → recurse right → visit node. Children are always handled before their parent, which makes postorder ideal when a parent's answer depends on its children — computing height (leaf heights bubble up), deleting a tree (delete children before parent), or aggregating subtree sums. A null node returns the identity value (0, false, [] etc.) before any work is done — this is the base case shared by all three orders.

Path Sum Easy
Given the root of a binary tree and integer targetSum, return true if the tree has a root-to-leaf path whose node values sum to targetSum.
function hasPathSum(root: TreeNode | null, target: number): boolean {
  if (!root) return false;
  // Base case: leaf node — check if this node's value equals the remaining target.
  if (!root.left && !root.right) return root.val === target;
  // Subtract current node's value and recurse on children.
  return hasPathSum(root.left, target - root.val) ||
         hasPathSum(root.right, target - root.val);
}

// Iterative DFS with explicit stack (preorder — right pushed first so left pops first)
function dfsIterative(root: TreeNode | null): number[] {
  const result: number[] = [], stack: TreeNode[] = [];
  if (root) stack.push(root);
  while (stack.length) {
    const node = stack.pop()!;
    result.push(node.val);
    if (node.right) stack.push(node.right); // push right first → left pops first (LIFO)
    if (node.left) stack.push(node.left);
  }
  return result;
}

How it works — Level-order BFS

Tree: 1 (level 0) / \ 2 3 (level 1) / \ / \ 4 5 6 7 (level 2) result = [] queue = [] Rule: snapshot queue.length BEFORE the inner loop so newly-enqueued children don't inflate the count for the current level.
① ──────────────────────────────────── Seed: enqueue root queue = [1] result = []
① Before the while-loop, push the root into the queue. The queue is non-empty, so the loop begins.
② ──────────────────────────────────── levelSize = 1 (snapshot queue.length) Dequeue 1 → level = [1] Enqueue children 2, 3 queue = [2, 3] result = [[1]]
② Snapshot levelSize = 1. Dequeue node 1, record its value. Node 1 has children 2 and 3 — enqueue both. The inner for-loop is done (i reached levelSize). Push level [1] into result.
③ ──────────────────────────────────── levelSize = 2 (snapshot queue.length) Dequeue 2 → level = [2]; enqueue 4, 5 Dequeue 3 → level = [2, 3]; enqueue 6, 7 queue = [4, 5, 6, 7] result = [[1], [2, 3]]
③ Snapshot levelSize = 2. First iteration: dequeue node 2, enqueue its children 4 and 5. Second iteration: dequeue node 3, enqueue its children 6 and 7. Inner loop ends. Push level [2, 3] into result.
④ ──────────────────────────────────── levelSize = 4 (snapshot queue.length) Dequeue 4, 5, 6, 7 → level = [4, 5, 6, 7] None have children → nothing enqueued queue = [] result = [[1], [2, 3], [4, 5, 6, 7]]
④ Snapshot levelSize = 4. All four leaf nodes are dequeued and recorded; none have children so nothing is enqueued. Push level [4, 5, 6, 7] into result. queue is now empty — the while-loop condition fails and BFS terminates. Return result. If root were null the early-return guard fires before we ever reach the loop.

Binary Tree Level Order Traversal Medium
Given the root of a binary tree, return the node values level by level (left to right) as an array of arrays.
function levelOrder(root: TreeNode | null): number[][] {
  if (!root) return [];
  const result: number[][] = [];
  const queue: TreeNode[] = [root];

  while (queue.length) {
    // Snapshot queue length BEFORE the loop — children added inside would change it mid-iteration.
    const levelSize = queue.length;
    const level: number[] = [];

    for (let i = 0; i < levelSize; i++) {
      const node = queue.shift()!;
      level.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }

    result.push(level);
  }

  return result;
}
Minimum Depth of Binary Tree Easy
Given the root of a binary tree, return the minimum depth — the number of nodes along the shortest path from root to the nearest leaf.
// BFS naturally finds the minimum depth: the first leaf we dequeue is the shallowest.
// DFS would need to explore all paths before knowing the minimum.
function minDepth(root: TreeNode | null): number {
  if (!root) return 0;
  const queue: [TreeNode, number][] = [[root, 1]];

  while (queue.length) {
    const [node, depth] = queue.shift()!;
    if (!node.left && !node.right) return depth; // first leaf = minimum depth
    if (node.left) queue.push([node.left, depth + 1]);
    if (node.right) queue.push([node.right, depth + 1]);
  }

  return 0;
}

How it works — BST search

Tree (searching for target = 6): 8 ← root / \ 3 10 / \ 1 6 ← target node / \ 4 7 BST invariant: all nodes in left subtree < node.val < all nodes in right subtree. Rule: target < node.val → go left; target > node.val → go right; equal → found.
① ──────────────────────────────────── curr = node(8) target 6 < 8 → go left
① Start at the root (value 8). Compare target 6 to 8: 6 < 8, so the target must be in the left subtree — move curr to node(3).
② ──────────────────────────────────── curr = node(3) target 6 > 3 → go right
② Compare target 6 to 3: 6 > 3, so the target must be in the right subtree — move curr to node(6).
③ ──────────────────────────────────── curr = node(6) target 6 == 6 → return node(6) ✓
③ curr.val equals the target — return curr. This is the found case. If instead curr became null before a match (e.g. searching for 9 in this tree: 9 > 8 → right to node(10), 9 < 10 → left, but node(10) has no left child → curr = null → return null), the function returns null signalling not-found.

Validate Binary Search Tree Medium
Given the root of a binary tree, determine if it is a valid BST. A valid BST requires ALL left descendants < node < ALL right descendants — not just direct children.
function searchBST(root: TreeNode | null, val: number): TreeNode | null {
  if (!root || root.val === val) return root;
  return val < root.val ? searchBST(root.left, val) : searchBST(root.right, val);
}

// Pass bounds down the tree — a node may be a local BST but violate the global invariant.
// E.g., a node with value 10 in the left subtree of 8 is invalid even if its own children are fine.
function isValidBST(
  root: TreeNode | null,
  min = -Infinity,
  max = Infinity
): boolean {
  if (!root) return true;
  if (root.val <= min || root.val >= max) return false; // strict inequalities: no equal values allowed
  return isValidBST(root.left, min, root.val) &&   // left subtree must be < current
         isValidBST(root.right, root.val, max);     // right subtree must be > current
}
Kth Smallest Element in a BST Medium
Given the root of a BST and an integer k, return the kth smallest value (1-indexed) among all node values.
// Iterative inorder: stops as soon as the kth element is found (no need to traverse all).
// Walk left as far as possible, then pop, then go right (classic inorder stack traversal).
function kthSmallest(root: TreeNode, k: number): number {
  const stack: TreeNode[] = [];
  let curr: TreeNode | null = root;

  while (curr || stack.length) {
    while (curr) { stack.push(curr); curr = curr.left; } // go as far left as possible
    curr = stack.pop()!;
    if (--k === 0) return curr.val; // kth element found
    curr = curr.right;
  }

  return -1;
}
Lowest Common Ancestor of a BST Easy
Given a BST and two nodes p and q, find their lowest common ancestor — the deepest node that has both as descendants (a node is a descendant of itself).
// Three cases — BST properties make this O(h) with no extra space:
// 1. Both p and q are in the left subtree  → recurse left
// 2. Both p and q are in the right subtree → recurse right
// 3. They diverge here (one on each side, or current node equals p or q) → current node is LCA
function lowestCommonAncestor(root: TreeNode, p: TreeNode, q: TreeNode): TreeNode {
  if (p.val < root.val && q.val < root.val) return lowestCommonAncestor(root.left!, p, q);
  if (p.val > root.val && q.val > root.val) return lowestCommonAncestor(root.right!, p, q);
  return root;
}

How it works — Graph BFS (shortest unweighted path)

Graph (undirected, unweighted): A ── B ── D | \── E +── C ── F ← target Adjacency: A:[B,C] B:[A,D,E] C:[A,F] D:[B] E:[B] F:[C] Goal: shortest path distance from A to F. Rule: mark visited at ENQUEUE time to prevent duplicate queue entries.
① ──────────────────────────────────── Seed: mark A visited, enqueue A queue = [A] visited = {A} dist = {A:0}
① Before the while-loop, mark the start node A as visited and push it with distance 0. Marking at enqueue time (not dequeue time) prevents the same node from being added to the queue more than once.
② ──────────────────────────────────── Dequeue A (dist=0) Neighbors of A: B (unvisited) → mark, enqueue dist=1 C (unvisited) → mark, enqueue dist=1 queue = [B, C] visited = {A, B, C} dist = {A:0, B:1, C:1}
② Dequeue A. Iterate over A's neighbors: B is not yet visited — mark it, set dist[B]=1, enqueue. C is not yet visited — mark it, set dist[C]=1, enqueue. Both level-1 nodes are now in the queue.
③ ──────────────────────────────────── Dequeue B (dist=1) Neighbors of B: A (visited, skip) D (unvisited) → mark, enqueue dist=2 E (unvisited) → mark, enqueue dist=2 queue = [C, D, E] visited = {A, B, C, D, E} dist = {A:0, B:1, C:1, D:2, E:2}
③ Dequeue B. Neighbor A is already visited — skip it to avoid revisiting. Neighbors D and E are unvisited — mark each, record dist=2, and enqueue. This is the neighbor-skipping mechanism that keeps BFS O(V + E).
④ ──────────────────────────────────── Dequeue C (dist=1) Neighbors of C: A (visited, skip) F (unvisited) → mark, enqueue dist=2 queue = [D, E, F] visited = {A, B, C, D, E, F} dist = {A:0, B:1, C:1, D:2, E:2, F:2}
④ Dequeue C. Neighbor A is visited — skip. Neighbor F is unvisited — mark it, set dist[F]=2, enqueue. F is the target node; because we enqueue at the moment of discovery, dist[F]=2 is already the shortest path.
⑤ ──────────────────────────────────── Dequeue D (dist=2): neighbors = [B] all visited, skip Dequeue E (dist=2): neighbors = [B] all visited, skip queue = [F] visited = {A, B, C, D, E, F} dist = {A:0, B:1, C:1, D:2, E:2, F:2}
⑤ Dequeue D: its only neighbor B is already visited — nothing is enqueued. Dequeue E: its only neighbor B is already visited — nothing is enqueued. Both are "dead-end" dequeues at level 2; they consume queue slots without expanding the frontier. F remains in the queue.
⑥ ──────────────────────────────────── Dequeue F (dist=2): F == target → return dist[F] = 2 ✓ queue = [] dist = {A:0, B:1, C:1, D:2, E:2, F:2}
⑥ Dequeue F. F is the target node — return dist[F] = 2, the shortest path length from A to F. Because we marked F visited and recorded its distance at enqueue time (block ④), the value is already correct and final. If no target had been found, the while-loop would exhaust the queue and exit; the caller returns -1 (or signals "unreachable") to indicate there is no path from the source to the target.

function bfs(graph: Map<string, string[]>, start: string): Map<string, number> {
  const dist = new Map<string, number>([[start, 0]]);
  const queue: string[] = [start];

  while (queue.length) {
    const node = queue.shift()!;
    for (const neighbor of graph.get(node) ?? []) {
      // Mark visited at ENQUEUE time (not dequeue) to prevent adding duplicates to the queue.
      if (!dist.has(neighbor)) {
        dist.set(neighbor, dist.get(node)! + 1);
        queue.push(neighbor);
      }
    }
  }

  return dist; // dist.get(target) = shortest path length
}

How it works — Graph DFS (recursive)

Graph (adjacency list): A → [B, C] B → [D, E] D → [F] E → [] C → [] F → [] Call dfs(A). visited = {} initially. Neighbors are processed left-to-right.
① ──────────────────────────────────── dfs(A) called visited = { A } next neighbor to explore: B (first of A's neighbors)
① dfs(A) is invoked. A is immediately marked visited (visited = { A }). We then iterate A's neighbors in order: [B, C]. B has not been visited, so we recurse into dfs(B) before touching C.
② ──────────────────────────────────── dfs(B) called visited = { A, B } next neighbor to explore: D (first of B's neighbors)
② dfs(B) runs. B is marked visited (visited = { A, B }). B's neighbors are [D, E]. D is unvisited, so we recurse into dfs(D).
③ ──────────────────────────────────── dfs(D) called visited = { A, B, D } D's only neighbor: F (unvisited) → recurse into dfs(F) dfs(F) called → visited = { A, B, D, F } F has no neighbors → dfs(F) returns
③ dfs(D) marks D visited (visited = { A, B, D }). D's neighbor list is [F]. F is unvisited, so we recurse into dfs(F). dfs(F) marks F visited (visited = { A, B, D, F }); F has no neighbors, so dfs(F) returns immediately.
④ ──────────────────────────────────── back in dfs(D): no more unvisited neighbors → dfs(D) returns (backtrack to dfs(B)) visited = { A, B, D, F }
④ Control returns to dfs(D). D has no remaining neighbors, so dfs(D) returns. The call stack unwinds back to dfs(B). visited remains { A, B, D, F }.
⑤ ──────────────────────────────────── back in dfs(B): next neighbor E (unvisited) dfs(E) called visited = { A, B, D, F, E } E has no neighbors → dfs(E) returns dfs(B) has no more neighbors → dfs(B) returns
⑤ Back in dfs(B), we continue to the next neighbor E. E is unvisited, so dfs(E) is called, marks E visited, finds no neighbors, and returns. dfs(B) now has no remaining neighbors and returns — another backtrack up to dfs(A).
⑥ ──────────────────────────────────── back in dfs(A): next neighbor C (unvisited) dfs(C) called visited = { A, B, D, F, E, C } C has no neighbors → dfs(C) returns dfs(A) has no more neighbors → dfs(A) returns
⑥ Back in dfs(A), the next neighbor C is unvisited. dfs(C) marks C visited, finds no neighbors, and returns. dfs(A) is now complete. Full DFS order: A → B → D → F → E → C. If the graph had a second component (unreachable from A), a top-level loop would call dfs on that component's start node next.
⑦ ──────────────────────────────────── skip-when-visited guard (cycle / shared neighbor scenario) suppose neighbor X is already in visited → the "if (!visited.has(X))" check is false → dfs(X) is NOT called; we move to the next neighbor
⑦ Whenever the loop over a node's neighbors reaches a node X that is already in visited, the guard (!visited.has(X)) is false and we simply skip X. This prevents infinite recursion in graphs with cycles and avoids double-counting nodes in disconnected-component problems.

Given n nodes (0 to n-1) and a list of undirected edges, return the number of connected components in the graph.
function countComponents(n: number, edges: number[][]): number {
  // Build adjacency list first (undirected: add both directions).
  const adj = new Map<number, number[]>();
  for (let i = 0; i < n; i++) adj.set(i, []);
  for (const [a, b] of edges) {
    adj.get(a)!.push(b);
    adj.get(b)!.push(a);
  }

  const visited = new Set<number>();
  let components = 0;

  function dfs(node: number): void {
    visited.add(node);
    for (const neighbor of adj.get(node) ?? []) {
      if (!visited.has(neighbor)) dfs(neighbor);
    }
  }

  // Iterate ALL nodes — graph may be disconnected, so not all nodes are reachable from node 0.
  for (let i = 0; i < n; i++) {
    if (!visited.has(i)) { dfs(i); components++; }
  }

  return components;
}
Course Schedule Medium

// 3-color DFS: 0 = unvisited, 1 = in current DFS path (cycle risk), 2 = fully processed (safe).
// State 1 on re-entry means we found a back edge → cycle exists.
function canFinish(numCourses: number, prerequisites: number[][]): boolean {
  const adj = Array.from({ length: numCourses }, () => [] as number[]);
  for (const [a, b] of prerequisites) adj[b].push(a); // b → a (take b before a)
  const state = new Array(numCourses).fill(0);

  function dfs(node: number): boolean {
    if (state[node] === 1) return false; // back edge: found a cycle
    if (state[node] === 2) return true;  // already fully processed: safe
    state[node] = 1; // mark: currently being explored
    for (const next of adj[node]) {
      if (!dfs(next)) return false;
    }
    state[node] = 2; // mark: done
    return true;
  }

  for (let i = 0; i < numCourses; i++) {
    if (!dfs(i)) return false;
  }
  return true;
}

How it works — Binary search

arr = [ 1, 3, 5, 7, 9, 11, 13, 15 ] target = 9 indices = 0 1 2 3 4 5 6 7 Loop runs while lo ≤ hi. mid = lo + Math.floor((hi - lo) / 2) (avoids integer overflow) Rules: arr[mid] < target → lo = mid + 1 (target must be in the right half) arr[mid] > target → hi = mid - 1 (target must be in the left half) arr[mid] == target → return mid (found) lo > hi → return -1 (not found — loop exits)
① ──────────────────────────────────── lo=0, hi=7 mid = 0 + floor((7-0)/2) = 3 arr[3] = 7 < 9 → lo = mid + 1 = 4
① Initial state: lo=0, hi=7. mid = 3. arr[3] = 7 which is less than the target 9, so the target must lie in the right half. We discard indices 0–3 by setting lo = 4. The loop condition lo ≤ hi (4 ≤ 7) is still true, so we iterate.
② ──────────────────────────────────── lo=4, hi=7 mid = 4 + floor((7-4)/2) = 5 arr[5] = 11 > 9 → hi = mid - 1 = 4
② lo=4, hi=7. mid = 5. arr[5] = 11 which is greater than the target 9, so the target must lie in the left half. We discard indices 5–7 by setting hi = 4. The loop condition lo ≤ hi (4 ≤ 4) is still true, so we iterate.
③ ──────────────────────────────────── lo=4, hi=4 mid = 4 + floor((4-4)/2) = 4 arr[4] = 9 == 9 → return 4 ✓
③ lo=4, hi=4. mid = 4. arr[4] = 9 equals the target. Return index 4. If instead the target were absent (e.g., target = 10), after the final pointer update (say lo = 5) the loop guard lo ≤ hi fails (5 > 4) and the loop exits, returning -1.

function binarySearch(nums: number[], target: number): number {
  let lo = 0, hi = nums.length - 1;
  while (lo <= hi) {
    const mid = lo + Math.floor((hi - lo) / 2); // avoids integer overflow (vs (lo+hi)/2)
    if (nums[mid] === target) return mid;
    else if (nums[mid] < target) lo = mid + 1;  // target in right half
    else hi = mid - 1;                           // target in left half
  }
  return -1;
}

    const mid = lo + Math.floor((hi - lo) / 2);

      else lo = mid + 1;
    } else { // right half [mid..hi] is sorted
      if (nums[mid] < target && target <= nums[hi]) lo = mid + 1; // target in sorted right half
      else hi = mid - 1;
    }
  }
  return -1;

How it works — Merge sort

arr = [ 5, 3, 8, 1 ] Merge sort is recursive: split the array in half until every sub-array has length 1 (base case), then merge sorted halves back together bottom-up. Complexity: O(n log n) time, O(n) auxiliary space (temporary merge buffer).
① ──────────────────────────────────── mergeSort([5, 3, 8, 1]) mid = 2 left = arr[0..1] = [5, 3] right = arr[2..3] = [8, 1] → recurse into mergeSort([5, 3]) first
① The input [5,3,8,1] has length 4, so we split at mid = 2: left half [5,3] and right half [8,1]. We recurse into mergeSort([5,3]) before touching the right half.
② ──────────────────────────────────── mergeSort([5, 3]) mid = 1 left = [5] ← length 1, base case, returns [5] right = [3] ← length 1, base case, returns [3]
② [5,3] splits into [5] and [3]. Both have length 1, which is the base case (arr.length ≤ 1 → return arr), so each returns immediately. We now merge [5] and [3].
③ ──────────────────────────────────── merge([5], [3]) i=0 j=0: left[0]=5 right[0]=3 → 3 < 5, take right → result=[3], j=1 j exhausted → append left remainder → result=[3, 5]
③ Merging [5] and [3]: compare left[0]=5 and right[0]=3. 3 < 5 so we take 3 from the right and advance j. j is now exhausted; we append the remaining left element 5. Sorted sub-result: [3, 5]. This is returned to the caller of mergeSort([5,3]).
④ ──────────────────────────────────── mergeSort([8, 1]) mid = 1 left = [8] ← length 1, base case, returns [8] right = [1] ← length 1, base case, returns [1]
④ Back up the call stack to mergeSort([8,1]). mid = 1, so left = [8] and right = [1]. Both have length 1 (base case: arr.length ≤ 1 → return arr), so each returns immediately. We now merge [8] and [1].
⑤ ──────────────────────────────────── merge([8], [1]) i=0 j=0: left[0]=8 right[0]=1 → 1 < 8 (left[i] ≤ right[j] is false) take right → result=[1], j=1 j exhausted → append left remainder → result=[1, 8]
⑤ Merging [8] and [1]: compare left[0]=8 and right[0]=1. The rule is take left when left[i] ≤ right[j]; here 8 > 1, so we take 1 from the right and advance j. j is now exhausted; we append the remaining left element 8. Sorted sub-result: [1, 8].
⑥ ──────────────────────────────────── merge([3, 5], [1, 8]) ← final merge i=0 j=0: 3 vs 1 → take 1, j=1, result=[1] i=0 j=1: 3 vs 8 → take 3, i=1, result=[1, 3] i=1 j=1: 5 vs 8 → take 5, i=2, result=[1, 3, 5] i exhausted → append right remainder [8] → result=[1, 3, 5, 8]
⑥ The final merge combines the two sorted halves [3,5] and [1,8]. left[i] ≤ right[j] decides which element to take: 3≤1 is false → take 1; 3≤8 → take 3; 5≤8 → take 5; left is exhausted so we append the remaining right element 8. Final sorted array: [1, 3, 5, 8].

function merge(left: number[], right: number[]): number[] {
  const result: number[] = [];
  let i = 0, j = 0;
  while (i < left.length && j < right.length) {
    if (left[i] <= right[j]) result.push(left[i++]);
    else result.push(right[j++]);

  return result.concat(left.slice(i)).concat(right.slice(j));

function nextGreaterElement(nums: number[]): number[] {

  const stack: number[] = []; // indices of elements waiting for their next greater

  for (let i = 0; i < nums.length; i++) {
    while (stack.length && nums[i] > nums[stack[stack.length - 1]]) {
      const idx = stack.pop()!;
      result[idx] = nums[i]; // nums[i] is the next greater element for the element at idx
    }
    stack.push(i); // current element hasn't found its answer yet
  }

  return result; // elements still in stack → no next greater → remain -1
}

House Robber Medium
You cannot rob two adjacent houses. Given an array of amounts, return the maximum you can rob without triggering alarms.
// dp[i] = max(skip house i → prev1, rob house i → prev2 + nums[i])
// Space-optimize: only need the last two values, not the full dp array.
function rob(nums: number[]): number {
  if (nums.length === 1) return nums[0];
  let prev2 = nums[0];                     // best without the previous house
  let prev1 = Math.max(nums[0], nums[1]);  // best including the previous house

  for (let i = 2; i < nums.length; i++) {
    const curr = Math.max(prev1, prev2 + nums[i]);
    prev2 = prev1;
    prev1 = curr;
  }

  return prev1;
}

Coin Change Medium
Given coin denominations and an amount, return the fewest number of coins needed to make up the amount. Return -1 if not possible.
// dp[i] = minimum coins to make amount i (Infinity = not yet reachable)
function coinChange(coins: number[], amount: number): number {
  const dp = new Array(amount + 1).fill(Infinity);
  dp[0] = 0; // base case: 0 coins needed to make amount 0

  for (let i = 1; i <= amount; i++) {
    for (const coin of coins) {
      if (coin <= i) {
        // Use one coin of this denomination + the optimal solution for the remainder.
        dp[i] = Math.min(dp[i], dp[i - coin] + 1);
      }
    }
  }

  return dp[amount] === Infinity ? -1 : dp[amount];
}

A robot at the top-left of an m × n grid can only move right or down. How many unique paths are there to the bottom-right corner?
function uniquePaths(m: number, n: number): number {
  // Initializing the entire array to 1 correctly sets both base cases:
  // - First row: only 1 way to reach any cell (move all the way right)
  // - First column: only 1 way to reach any cell (move all the way down)
  const dp = Array.from({ length: m }, () => new Array(n).fill(1));
  for (let i = 1; i < m; i++) {
    for (let j = 1; j < n; j++) {
      dp[i][j] = dp[i-1][j] + dp[i][j-1]; // paths from above + paths from left
    }
  }
  return dp[m-1][n-1];
}
Longest Common Subsequence Medium
Given two strings, return the length of their longest common subsequence. A subsequence does not need to be contiguous.
// dp[i][j] answers: "LCS of text1[0..i-1] and text2[0..j-1]"
// Indices shifted by 1 so dp[0][*] and dp[*][0] are naturally 0 (empty string base cases).
function lcs(text1: string, text2: string): number {
  const m = text1.length, n = text2.length;
  const dp = Array.from({ length: m + 1 }, () => new Array(n + 1).fill(0));

  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (text1[i-1] === text2[j-1]) {
        dp[i][j] = dp[i-1][j-1] + 1;               // chars match: extend the diagonal LCS by 1
      } else {
        dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]); // skip a char from either string, take the best
      }
    }
  }

  return dp[m][n];
}

function minDistance(word1: string, word2: string): number {
  const m = word1.length, n = word2.length;
  // Base cases: converting to/from empty string requires inserting/deleting all characters.
  // dp[i][0] = i (delete i chars from word1); dp[0][j] = j (insert j chars).
  const dp = Array.from({ length: m + 1 }, (_, i) =>
    Array.from({ length: n + 1 }, (_, j) => i === 0 ? j : j === 0 ? i : 0)

    for (let j = 1; j <= n; j++) {
      if (word1[i-1] === word2[j-1]) {
        dp[i][j] = dp[i-1][j-1];       // chars match: no operation needed
      } else {
        dp[i][j] = 1 + Math.min(
          dp[i-1][j],    // delete word1[i-1]  (come from cell above)

          dp[i-1][j-1]   // replace word1[i-1] with word2[j-1] (come from diagonal)
        );
      }
    }
  }

  return dp[m][n];

How it works — The universal template

  1. Base case: When is the current path a valid complete solution? Add to results.
  2. Loop over choices: For each valid choice at the current step...
  3. Choose: Add the choice to the current path.
  4. Recurse: Call backtrack with updated state.
  5. Unchoose: Remove the choice (restore state for the next iteration).
  6. Prune early: Skip choices that cannot lead to a valid solution (e.g., sum > target).

Subsets Medium
Given an integer array of unique elements, return all possible subsets (the power set). The solution set must not contain duplicate subsets.
function subsets(nums: number[]): number[][] {
  const results: number[][] = [];

  function bt(start: number, path: number[]): void {
    // Record [...path] at EVERY call (not just leaves) — this generates all 2^n subsets,
    // since every intermediate prefix is itself a valid subset.
    results.push([...path]);

    for (let i = start; i < nums.length; i++) {
      path.push(nums[i]);      // CHOOSE
      bt(i + 1, path);         // RECURSE — i+1 prevents re-using elements
      path.pop();              // UNCHOOSE (backtrack)
    }
  }

  bt(0, []);
  return results;
}

// Unlike subsets, permutations can use any remaining element — not just elements after the current index.
// That's why we use a `used[]` boolean array instead of a `start` index.
function permute(nums: number[]): number[][] {
  const results: number[][] = [];
  const used = new Array(nums.length).fill(false);

  function bt(path: number[]): void {
    if (path.length === nums.length) { results.push([...path]); return; }

    for (let i = 0; i < nums.length; i++) {
      if (used[i]) continue;
      used[i] = true;
      path.push(nums[i]);   // CHOOSE
      bt(path);             // RECURSE
      path.pop();           // UNCHOOSE
      used[i] = false;
    }
  }

  bt([]);
  return results;
}
Combination Sum Medium
Given distinct candidates and a target, return all unique combinations that sum to target. The same number may be chosen unlimited times.
function combinationSum(candidates: number[], target: number): number[][] {
  const results: number[][] = [];

  function bt(start: number, path: number[], remaining: number): void {
    if (remaining === 0) { results.push([...path]); return; }
    if (remaining < 0) return; // PRUNE: overshooting target — no valid path down here

    for (let i = start; i < candidates.length; i++) {
      path.push(candidates[i]);
      bt(i, path, remaining - candidates[i]); // pass i (not i+1): allows reusing the same element
      path.pop();
    }
  }

  bt(0, [], target);
  return results;
}
